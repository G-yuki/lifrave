Lifrave プロジェクト引き継ぎ情報
アプリ概要
アプリ名：Lifrave（仮）
コンセプト：夫婦・カップルの「やりたいこと」をAIが提案し、
           一緒に消化していくバケットリストアプリ。
           ログが溜まるにつれ思い出アルバムの作成準備ができる。
テーマ：人生の思い出を形にする
ターゲット：夫婦・カップル
プラットフォーム：Webアプリ（将来的にアプリ化）

技術スタック（確定）
フロントエンド：React + TypeScript + Vite + Tailwind CSS v4
DB：Firebase Firestore
認証：Firebase Auth（Googleログイン）
APIサーバー：Firebase Cloud Functions（Claude API・Google Maps APIキー管理）
デプロイ：Firebase Hosting
外部API：
  - Claude API（リスト生成・思い出生成）
  - Google Maps Places API（スワイプカードのサムネイル・評価取得）
パッケージ：
  - firebase
  - react-router-dom
  - @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities（長押しドラッグ並び替え）

Firestoreコレクション構造
/users/{uid}
  - displayName: string
  - pairId: string | null
  - createdAt: timestamp

/pairs/{pairId}
  - members: [uid, uid]
  - inviteToken: string
  - isActive: boolean
  - createdAt: timestamp
  - hearing: {
      genres: string[]          // 好きな体験タイプ
      prefecture: string        // 都道府県
      range: string             // "county" | "neighbor" | "anywhere"
      children: string          // "none" | "infant" | "child" | "planned"
      transport: string         // "transit" | "car" | "both"
      budget: string            // "3000" | "5000" | "10000" | "30000" | "any"
      indoor: string            // "outdoor" | "indoor" | "both"
      freetext: string
    }

/pairs/{pairId}/items/{itemId}
  - title: string               // 15文字以内
  - category: string            // "おでかけ"|"映画"|"本"|"ゲーム"|"食事"|"音楽"|"スポーツ"|"その他"
  - type: string                // "outdoor" | "indoor"
  - difficulty: string          // "easy" | "special"
  - status: string              // "unread" | "todo" | "done"
  - isWant: boolean             // やりたい★フラグ
  - rating: number | null       // 1〜5
  - memo: string | null         // 100文字以内
  - completedAt: timestamp | null
  - createdAt: timestamp
  - placeId: string | null      // Google Maps Place ID（おでかけ系のみ）
  - placeName: string | null
  - placeRating: number | null
  - placePhotoRef: string | null

主要仕様（確定）
ヒアリング（6ステップ）
STEP 1：どんな体験が好き？（複数選択）
  自然・アウトドア / グルメ・食べ歩き / アート・文化 /
  音楽・ライブ / スポーツ / 映画・ドラマ /
  本・読書 / ゲーム・カフェ / テーマパーク / 温泉・スパ

STEP 2：主に活動するエリアは？
  都道府県プルダウン → 県内中心 / 隣県まで / 全国OK

STEP 3：お子さまはいますか？
  いない・予定なし / 乳幼児あり / 小学生以上あり / 今後予定あり

STEP 4：移動手段は？
  電車・バスのみ / 車あり / 両方使う

STEP 5：予算感は？（1回あたり・ふたり／家族合計）
  〜3,000円 / 〜5,000円 / 〜10,000円 / 〜30,000円 / 気にしない

STEP 6：屋内・屋外の好みは？
  屋外が好き / 屋内が好き / どちらでもOK
  + 自由入力（任意・100文字）
AI初期リスト生成
- Claude APIを1回コール（Cloud Functions経由）
- ヒアリング結果をプロンプトに埋め込み50件生成
- 出力フォーマット（JSON）：
  { title: string, category: string, type: string, difficulty: string }
- トークン試算：入力400 + 出力1500 = 約1900トークン（約3〜4円/回）
スワイプ選択
- 右スワイプ：採用（todo）
- 左スワイプ：除外
- 上スワイプ：やりたい★（isWant: true）
- おでかけ系のみGoogle Maps Places APIでサムネイル・評価取得
- カード表示項目：サムネイル / タイトル / カテゴリタグ / Google Maps評価 / 概要
提案ロジック（ルールベース・AI不使用）
① isWant:true のアイテムを優先
② なければ未着手からランダム抽出
③ 直近30日以内に完了したカテゴリは除外
④ 1〜3件を提示
⑤ 「もう一度」は3回まで制限
リスト管理
- メモ：100文字以内（UIカウンター表示 + Firestore側バリデーション）
- 完了チェック取り消し：時間制限なし
- 検索：フロント側でフィルタリング（Firestoreクエリ不使用）
- 並び替え：@dnd-kit 長押しドラッグ
- 完了済みは下部に折りたたみ表示
思い出生成
- 期間選択：1〜6ヶ月（1ヶ月刻み）
- 生成物①：年間レポート（テキスト）
- 生成物②：プレイリストプロンプト → Sunoへの案内
- 生成物③：アルバム表紙プロンプト → Geminiへの案内（gemini.google.com）
- Claude API 1コールで3種を同時生成
Google Maps API
- 使用API：Place Photos（Pro: 月5,000回無料）+ Place Details（Essentials: 月10,000回無料）
- おでかけ系カテゴリのみAPI呼び出し
- 取得した情報はFirestoreにキャッシュ（毎回API呼び出しをしない）

Phase 1 Step 1 の作業内容（まずここから）
powershell# 1. 作業ディレクトリに移動
cd {任意の作業ディレクトリ}

# 2. Viteでプロジェクト作成
npm create vite@latest lifrave -- --template react-ts

# 3. ディレクトリ移動・依存インストール
cd lifrave
npm install

# 4. 追加パッケージを一括インストール
npm install firebase react-router-dom @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities

# 5. Tailwind CSS v4インストール
npm install -D tailwindcss @tailwindcss/postcss postcss autoprefixer
